# Teaching: Gated MLPs in Transformer Feed-Forward Blocks

## Progress: 0/11 concepts confirmed

### The Problem
- [ ] A standard transformer MLP expands hidden activations, applies a nonlinearity, then projects back down.
- [ ] The expansion dimension gives the MLP more channel capacity than the residual hidden size alone.
- [ ] A plain activation such as GELU transforms values, while a gated MLP additionally learns which expanded channels should pass through.

### The Solution
- [ ] A gated MLP computes two expanded branches from the same input: a gate projection and an up projection.
- [ ] The gate projection produces learned per-token, per-channel gates after an activation such as SiLU or GELU.
- [ ] The up projection produces the candidate expanded features that carry content forward.
- [ ] The activated gate branch is multiplied elementwise with the up branch, so the gate modulates the candidate features.
- [ ] The down projection maps the gated expanded representation back to the model hidden size so it can rejoin the residual stream.

### Broader Context
- [ ] Common variants include GLU, GEGLU, and SwiGLU, mainly differing in the gate activation function.
- [ ] Gate and up projections are usually column-parallel in tensor parallelism because they split expanded output channels.
- [ ] The down projection is usually row-parallel in tensor parallelism because it consumes the expanded channels and sums partial outputs.
